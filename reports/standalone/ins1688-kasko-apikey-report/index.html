Баг: создание КАСКО-заявки через /app/new

INS-3239 — API /app/new возвращает result: false с ошибкой «Только для заявок с типом продукта КАСКО» при отправке productType: "Kasko". При этом заявка реально создаётся в БД с ProductTypeId=2, но ApplicationId не возвращается. За 20.04 создано 8 КАСКО-заявок через этот ключ — все в БД, ни одна не вернула успешный ответ.

ОСАОГО через тот же ключ создаётся корректно. Старый ключ f273ada6ac954e7896cd23b6a10f2e72 работает без ошибок.

Причина: баг в серверном коде ApplicationService.Create. Илья проверил на тесте — даже после выравнивания всех настроек ошибка сохраняется. Вероятно, привязка к конкретному ApiKeyId в коде.
