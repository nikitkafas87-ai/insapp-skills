Замечания

Про `KaskoProcessTypeId` В предыдущей версии отчета колонка была показана неудачно. В новой версии `KaskoProcessTypeId` выведен только для подключенного KASKO-ключа, по которому работает фича. Для связок OSAGO -> KASKO валидируются конкретные поля: `KaskoFromOsagoApiKeyId`, активность ключей, `SupportedProductTypesJson`, KASKO `KaskoProcessTypeId`, `LandingUrl` на KASKO-ключе и фактическое создание дочерних заявок.
Сохраняется риск по ГПБ На OSAGO-ключе `d9d1d57c-0288-48d9-87a9-e22f760154f0` ранее найдено 39 `ErrorLogs`: `Неподдерживаемый KaskoProcessType`. По проверенным примерам дочерние КАСКО не создавались.

Методика

  • Дочерние КАСКО искались по `ProductTypeId = 2` и `ApplicationTypeId = 9`.
  • Период: `2026-05-19 00:00:00.000 +03:00` <= `Created` < `2026-05-23 00:00:00.000 +03:00`.
  • Источники: `InsappCoreProd` (`Applications`, `Offers`, `ApplicationStatuses`, `PartnerApiKeys`, `Partners`, `Consents`) и `InsappLogProd` (`BusinessLogs`, `ErrorLogs`).
  • Все запросы read-only SELECT; полные GUID не сокращались.
